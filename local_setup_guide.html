A SFT & Reasoning Training Datasets

MetaMathQA
395,000 examples · math-genie/MetaMathQA
✓ PRIMARY — Use this first ⭐ Best SFT dataset
What it is: Mathematical Reasoning Augmented Training dataset. Takes GSM8K and MATH problems and reformulates each one 3–7 different ways: forward reasoning, backward reasoning, rephrased versions, and FOBAR (find-the-other-answer variants). Every example has a complete step-by-step solution.

Why it's the best for your task: You don't just get one path to the answer — you get multiple problem formulations per concept. This forces the model to learn the reasoning pattern, not just memorise a specific problem's solution path. GSM8K accuracy jumps are consistently reported at +8-15% over GSM8K-only SFT when MetaMathQA is used.

How to use: Sample 30,000 examples stratified across all subtypes (GSM_MATH, GSM_ART, GSM_Rephrased, GSM_SV, MATH_CoT, MATH_Rephrased). Sort by difficulty for curriculum. This alone will be your primary SFT dataset.
Orca-Math-200K
200,000 examples · microsoft/orca-math-word-problems-200k
✓ Use — 10K curated sample
What it is: Microsoft Research dataset of 200K word problems with GPT-4-Turbo generated solutions. High-quality, diverse problem types — rate/ratio, percentage, geometry, mixture, age problems. Solutions are detailed, multi-step, well-structured.

Why to use it: The GPT-4 solutions are significantly more coherent than many open-source math datasets. Problem variety is high — many problem types GSM8K doesn't cover. Complements MetaMathQA well since it covers different linguistic styles.

How to use: Sample 10,000 examples. Filter by solution length (>150 chars). This adds problem-type diversity to your SFT training set.
GSM8K (train)
7,473 problems · openai/gsm8k
✓ Use all of it
Role: Core SFT + ORM baseline The benchmark dataset itself. Every training example here directly improves the GSM8K test score (same distribution). Integer answers enable the cleanest possible rule-based ORM — no neural reward model needed, no false positives. Use all 7,473 train examples. Already has built-in step reasoning in the answer field (split on "####"). This is your ground truth for the ORM reward during RAFT.
Open-CoT
Varies by subset · kaist-ai/open-cot
⚡ Use select subsets
Role: Reasoning format diversity Collection of chain-of-thought demonstrations from various sources. Quality is inconsistent — some subsets are excellent (GSM8K CoT, MATH CoT), others are low quality. Use only: gsm8k_cot, math_cot, and strategyqa_cot subsets. The StrategyQA CoT subset is valuable — it's your best training signal for StrategyQA benchmark transfer since no other provided dataset covers multi-hop reasoning.
HotpotQA
90,447 train · hotpot_qa
⚡ 5K sample only
Role: Multi-hop reasoning for StrategyQA Multi-step Wikipedia question answering requiring 2+ reasoning hops. StrategyQA (your benchmark) is a binary yes/no multi-hop dataset. HotpotQA is the best available proxy for training multi-hop reasoning transfer. Sample 5,000 examples with supporting facts. Convert to CoT format: "Fact 1... Fact 2... Therefore: [answer]". This is how your model learns to chain evidence — critical for StrategyQA.
UltraInteract
Large · openbmb/UltraInteract_sft
⚡ Math subset only
Role: Complex multi-step SFT Multi-turn reasoning traces with tree-structured search. Contains math, coding, and logic problem-solving interactions. Use only the "math" task subset. These examples tend to be harder problems with more elaborate reasoning chains — useful for the "hard" tier of your curriculum. Sample 3,000 math examples. The multi-turn structure shows self-correction naturally.
FLAN subset
Massive · Muennighoff/flan
✗ Skip for this project
Role: General instruction — not needed FLAN is an instruction-tuning mega-dataset covering hundreds of NLP tasks. Gemma 4 E4B is already instruction-tuned. Adding FLAN will dilute your math/reasoning focus and potentially hurt GSM8K and StrategyQA scores by shifting the model toward generic instruction following. MMLU doesn't need FLAN either — its knowledge comes from pretraining. Skip this entirely.

Script 2 — Format Everything to Unified CoT

scripts/02_format_data.pypython
"""
Format all datasets → unified CoT JSONL format.
Output: data/processed/all_formatted.jsonl
"""
import json, random, re
from datasets import load_dataset
from pathlib import Path

random.seed(42)
Path("data/processed").mkdir(exist_ok=True)

# ── The universal template every formatter must produce ───────
# Every example MUST have: prompt, completion, answer, task, source, difficulty
# completion MUST contain <think>...</think><answer>...</answer>

def fmt_gsm8k(ex):
    parts   = ex["answer"].split("####")
    steps   = parts[0].strip()
    answer  = parts[1].strip() if len(parts) > 1 else ""
    n_words = len(ex["question"].split())
    return {
        "prompt": f"Solve step-by-step:\n{ex['question']}",
        "completion": f"<think>\n{steps}\n</think>\n<answer>{answer}</answer>",
        "answer": answer, "task": "math",
        "source": "gsm8k", "difficulty": min(n_words/80, 1.0),
    }

def fmt_metamath(ex):
    # MetaMathQA has: query (question), response (full solution)
    query    = ex.get("query", "").strip()
    response = ex.get("response", "").strip()
    # Extract final answer — usually "The answer is \boxed{X}" or "#### X"
    ans_m = re.search(r'(?:The answer is|\\boxed{|####\s*)([^}.\n]+)', response)
    answer = ans_m.group(1).strip() if ans_m else ""
    # Difficulty: longer responses = harder reasoning chain
    difficulty = min(len(response.split()) / 300, 1.0)
    return {
        "prompt": f"Solve step-by-step:\n{query}",
        "completion": f"<think>\n{response}\n</think>\n<answer>{answer}</answer>",
        "answer": answer, "task": "math",
        "source": "metamath", "difficulty": difficulty,
    }

def fmt_orca_math(ex):
    # Orca-Math has: question, answer (full solution with final number)
    question = ex.get("question", "").strip()
    solution = ex.get("answer", "").strip()
    # Final answer is usually the last number in the solution
    nums = re.findall(r'\b\d+\.?\d*\b', solution)
    final_ans = nums[-1] if nums else ""
    return {
        "prompt": f"Solve step-by-step:\n{question}",
        "completion": f"<think>\n{solution}\n</think>\n<answer>{final_ans}</answer>",
        "answer": final_ans, "task": "math",
        "source": "orca_math", "difficulty": min(len(solution)/400, 1.0),
    }

def fmt_hotpotqa(ex):
    # HotpotQA: question, answer, supporting_facts (list of [title, sent_id])
    question = ex.get("question", "")
    answer   = ex.get("answer", "")
    # Build reasoning from supporting facts (context contains the actual sentences)
    context  = ex.get("context", {})
    titles   = context.get("title", [])
    sents    = context.get("sentences", [])
    # Simple: join supporting document titles as reasoning hints
    reasoning = f"This requires checking multiple sources.\nSources: {', '.join(titles[:3])}.\nBased on these facts: {answer}"
    return {
        "prompt": f"Answer this step-by-step using evidence:\n{question}",
        "completion": f"<think>\n{reasoning}\n</think>\n<answer>{answer}</answer>",
        "answer": answer.lower(), "task": "qa",
        "source": "hotpotqa", "difficulty": 0.65,
    }

# ── Load and format each dataset ─────────────────────────────
all_examples = []

# 1. GSM8K — use ALL training examples
print("Formatting GSM8K...")
gsm = load_dataset("gsm8k", "main", cache_dir="data/raw")
gsm_formatted = [fmt_gsm8k(ex) for ex in gsm["train"]]
all_examples.extend(gsm_formatted)
print(f"  → {len(gsm_formatted)} GSM8K examples")

# 2. MetaMathQA — sample 30K across subtypes
print("Formatting MetaMathQA...")
meta = load_dataset("meta-math/MetaMathQA", cache_dir="data/raw")
meta_formatted = []
subtypes = {}  # Count by type for stratified sampling
for ex in meta["train"]:
    qtype = ex.get("type", "unknown")
    subtypes[qtype] = subtypes.get(qtype, [])
    subtypes[qtype].append(fmt_metamath(ex))

samples_per_type = 5000  # 6 types × 5K = 30K
for qtype, examples in subtypes.items():
    sampled = random.sample(examples, min(samples_per_type, len(examples)))
    meta_formatted.extend(sampled)
    print(f"  {qtype}: {len(sampled)} samples")

all_examples.extend(meta_formatted)
print(f"  → {len(meta_formatted)} MetaMathQA examples")

# 3. Orca-Math — sample 10K
print("Formatting Orca-Math...")
orca = load_dataset("microsoft/orca-math-word-problems-200k", cache_dir="data/raw")
orca_all = [fmt_orca_math(ex) for ex in orca["train"]
            if len(ex.get("answer","")) > 100]  # Quality filter
orca_sampled = random.sample(orca_all, 10_000)
all_examples.extend(orca_sampled)
print(f"  → {len(orca_sampled)} Orca-Math examples")

# 4. HotpotQA — 5K sample (for StrategyQA transfer)
print("Formatting HotpotQA...")
hotpot = load_dataset("hotpot_qa", "distractor", cache_dir="data/raw")
hotpot_formatted = [fmt_hotpotqa(ex) for ex in hotpot["train"]]
hotpot_sampled = random.sample(hotpot_formatted, 5_000)
all_examples.extend(hotpot_sampled)
print(f"  → {len(hotpot_sampled)} HotpotQA examples")

# ── Save raw formatted (before Qwen filtering) ────────────────
out_path = "data/processed/all_formatted.jsonl"
with open(out_path, "w") as f:
    for ex in all_examples:
        f.write(json.dumps(ex, ensure_ascii=False) + "\n")

print(f"\n✅ Total formatted: {len(all_examples)} examples")
print(f"Saved to {out_path}")
print("Next: run 03_qwen_filter.py to quality-score with Qwen 14B")
Qwen 14B Teacher Role

Ollama as
Curriculum Architect

Qwen 14B via Ollama plays three roles: quality judge (filter bad examples), difficulty estimator (order curriculum), and CoT generator (improve weak examples). It runs before any training starts — completely sequential.

📐
Why Qwen 14B specifically? Qwen 2.5 14B has exceptional math reasoning benchmarks — it scores ~82% on GSM8K out of the box. This means when it rates a training example as "low quality", it's almost certainly right. Gemma 4 E4B (your student) will be learning from examples that a much stronger math model has validated. This quality signal is the hidden advantage of having Qwen locally.

Ollama Python API Setup

scripts/ollama_client.pypython
"""Reusable Ollama client for Qwen 14B calls."""
import requests, json, time
from config import OLLAMA_URL, QWEN_MODEL

def qwen_chat(prompt: str, temperature: float = 0.1,
              max_tokens: int = 256, timeout: int = 120) -> str:
    """Send a prompt to Qwen 14B via Ollama and return the response."""
    payload = {
        "model": QWEN_MODEL,
        "messages": [{"role": "user", "content": prompt}],
        "stream": False,
        "options": {"temperature": temperature, "num_predict": max_tokens},
    }
    try:
        r = requests.post(f"{OLLAMA_URL}/api/chat",
                          json=payload, timeout=timeout)
        r.raise_for_status()
        return r.json()["message"]["content"].strip()
    except Exception as e:
        print(f"Ollama error: {e}")
        return ""

def qwen_rate_quality(prompt: str, completion: str) -> float:
    """Ask Qwen to rate a training example 1-10 for reasoning quality.
    Returns float between 0.1 and 1.0."""
    rating_prompt = f"""You are a math education expert. Rate this math solution for REASONING QUALITY only.

Problem: {prompt[:300]}

Solution (first 400 chars): {completion[:400]}

Rate ONLY the quality of the step-by-step reasoning process (not whether the final answer is correct).
Consider: Are steps explicit? Is logic sound? Are calculations shown? Is reasoning complete?

Respond with ONLY a single integer 1-10. Nothing else. Just the number."""

    response = qwen_chat(rating_prompt, temperature=0.0, max_tokens=5)
    try:
        score = int(response.strip()[0])
        return max(1, min(10, score)) / 10.0
    except:
        return 0.5  # default if parse fails

def qwen_estimate_difficulty(problem: str) -> float:
    """Estimate how hard a problem is for a ~4B parameter model. 0.0=easy, 1.0=hard"""
    diff_prompt = f"""Rate the difficulty of this math/reasoning problem for a small 4B language model.

Problem: {problem[:250]}

Consider: number of steps needed, mathematical complexity, multi-step reasoning required.

Respond with ONLY a single integer 1-10 (1=trivially easy, 10=extremely hard). Just the number."""

    response = qwen_chat(diff_prompt, temperature=0.0, max_tokens=5)
    try:
        score = int(response.strip()[0])
        return max(1, min(10, score)) / 10.0
    except:
        return 0.5

def qwen_improve_cot(problem: str, weak_cot: str, answer: str) -> str:
    """If a CoT is weak (score < 5), ask Qwen to rewrite it better."""
    improve_prompt = f"""Rewrite this solution to be clearer and more explicit, step by step.

Problem: {problem[:300]}
Original solution (may be incomplete): {weak_cot[:300]}
Correct final answer: {answer}

Write a clear, explicit step-by-step solution. Show every calculation.
Use format:
Step 1: [action]
Step 2: [action]
...
Therefore: [final answer]"""

    return qwen_chat(improve_prompt, temperature=0.3, max_tokens=400)

"""
Quality-filter formatted examples using Qwen 14B.
Rates each example 1-10, estimates difficulty, optionally improves weak CoTs.
Input:  data/processed/all_formatted.jsonl
Output: data/splits/train.jsonl (sorted by difficulty, quality >= 6)

Runtime: ~6-8 hours for 55K examples at ~0.5s/example
Start in morning, check in evening.
"""
import json, time, random
from pathlib import Path
from ollama_client import qwen_rate_quality, qwen_estimate_difficulty, qwen_improve_cot

Path("data/splits").mkdir(exist_ok=True)

# ── Load all formatted examples ────────────────────────────────
with open("data/processed/all_formatted.jsonl") as f:
    all_examples = [json.loads(l) for l in f]

print(f"Loaded {len(all_examples)} examples for Qwen filtering")
print("Make sure Ollama is running: ollama serve")
print("Make sure Qwen is pulled: ollama pull qwen2.5:14b-instruct-q4_K_M\n")

# ── Process with progress saving every 500 examples ──────────
# Critical: save progress so you can resume if interrupted
progress_file = "data/processed/qwen_progress.jsonl"
scored = []

# Resume from where we left off
if Path(progress_file).exists():
    with open(progress_file) as f:
        scored = [json.loads(l) for l in f]
    print(f"Resuming from {len(scored)} already processed")

already_done = {hash(ex["prompt"][:50]) for ex in scored}
remaining    = [ex for ex in all_examples
               if hash(ex["prompt"][:50]) not in already_done]

with open(progress_file, "a") as prog_file:
    for i, ex in enumerate(remaining):
        t_start = time.time()

        # 1. Rate reasoning quality
        quality = qwen_rate_quality(ex["prompt"], ex["completion"])

        # 2. Estimate difficulty (use existing difficulty if available)
        if ex.get("difficulty", 0) == 0.5:  # Generic default → get Qwen estimate
            difficulty = qwen_estimate_difficulty(ex["prompt"])
        else:
            difficulty = ex["difficulty"]  # Already computed from length heuristic

        # 3. Optionally improve weak CoTs from good sources
        improved_cot = ex["completion"]
        if quality < 0.5 and ex["source"] in ["gsm8k", "metamath"]:
            # Problem is good (well-known source) but CoT is weak → improve it
            better = qwen_improve_cot(ex["prompt"], ex["completion"], ex["answer"])
            if better and len(better) > 50:
                improved_cot = f"<think>\n{better}\n</think>\n<answer>{ex['answer']}</answer>"
                quality = 0.75  # Bump quality score after improvement

        result = {**ex,
                  "quality_score": quality,
                  "difficulty": difficulty,
                  "completion": improved_cot}
        scored.append(result)
        prog_file.write(json.dumps(result) + "\n")
        prog_file.flush()

        elapsed = time.time() - t_start
        if (i+1) % 100 == 0:
            pct = (i+1) / len(remaining) * 100
            hrs_left = elapsed * (len(remaining) - i - 1) / 3600
            print(f"[{i+1}/{len(remaining)}] {pct:.0f}% · {hrs_left:.1f}h remaining")

# ── Filter: keep quality >= 0.6, drop empty answers ──────────
filtered = [ex for ex in scored
            if ex["quality_score"] >= 0.6
            and len(ex["answer"].strip()) > 0
            and "<think>" in ex["completion"]
            and "<answer>" in ex["completion"]]

# ── Sort by difficulty (curriculum learning: easy first) ──────
filtered.sort(key=lambda x: x["difficulty"])

# Hold out last 500 as internal val set
train_set = filtered[:len(filtered)-500]
val_set   = filtered[len(filtered)-500:]

with open("data/splits/train.jsonl", "w") as f:
    for ex in train_set:
        f.write(json.dumps(ex, ensure_ascii=False) + "\n")

with open("data/splits/valid.jsonl", "w") as f:
    for ex in val_set:
        f.write(json.dumps(ex, ensure_ascii=False) + "\n")

print(f"\n✅ Quality filtering complete!")
print(f"   Kept: {len(filtered)} / {len(scored)} ({len(filtered)/len(scored)*100:.0f}%)")
print(f"   Train: {len(train_set)} · Val: {len(val_set)}")
print("Next: quit Ollama, then run SFT training")
print("  → bash scripts/04_sft_train.sh")

Phase 3 — Local RL

RAFT Iterations
Reinforcement Learning on 24GB

RAFT (Reward rAnked Fine-Tuning) is your local RL mechanism. Generate K=4 solutions per problem, score by rule-based ORM, fine-tune on winners only. Three iterations. No second model needed.

Why RAFT instead of GRPO?
GRPO needs 2× model copies in memory simultaneously. RAFT only needs one. RAFT paper showed matching PPO performance with simpler implementation. K=4 (not 8) because that's what fits in 24GB for Gemma 4 E4B. Each RAFT iteration is just: generate → score → train on winners.
Expected gains per RAFT iteration
Iteration 1: +3-5% GSM8K over SFT baseline. Iteration 2: +2-3% additional. Iteration 3: +1-2% additional. Total: +6-10% over SFT, which already beat baseline by ~15-20%. Combined gain over raw baseline: ~21-30%.
scripts/06_raft_loop.pypython
"""
RAFT: Local Reinforcement Learning for Gemma 4 E4B
One script runs all 3 iterations sequentially.
Usage: python scripts/06_raft_loop.py --start-iter 1
Runtime: ~3-4 hours per iteration = ~10-12 hours total
"""
import json, re, argparse, subprocess, random
from pathlib import Path
from mlx_lm import load, generate
from config import BASE_MODEL, RAFT_K, RAFT_ITERS

def orm_score(completion: str, ground_truth: str, task: str = "math") -> float:
    """Rule-based Outcome Reward Model. 1.0=correct, 0.0=wrong.
    For math: exact number match. For MCQ: letter match."""
    m = re.search(r'<answer>\s*(.*?)\s*</answer>', completion, re.DOTALL)
    pred = m.group(1).strip() if m else ""
    pred = re.sub(r'[,$]', '', pred).strip()
    gt   = re.sub(r'[,$]', '', ground_truth).strip()

    if task == "math":
        try: return 1.0 if abs(float(pred) - float(gt)) < 0.01 else 0.0
        except: return 1.0 if pred == gt else 0.0
    else:
        pred_l = pred.upper()[:1]
        gt_l   = gt.upper()[:1]
        return 1.0 if pred_l == gt_l else 0.0

def format_reward(completion: str) -> float:
    """Small reward for using correct <think><answer> format."""
    has_think  = "<think>" in completion and "</think>" in completion
    has_answer = "<answer>" in completion and "</answer>" in completion
    has_steps  = completion.count("\n") >= 2
    return 0.1 if (has_think and has_answer and has_steps) else 0.0

def run_raft_iteration(iteration: int, model, tokenizer, train_data: list):
    print(f"\n{'='*50}")
    print(f"RAFT Iteration {iteration}/{RAFT_ITERS}")
    print(f"{'='*50}")

    raft_winners = []   # (prompt, best_completion) pairs
    stats = {"correct_k1":0, "correct_any":0, "total":0}

    # Sample 2000 problems per RAFT iteration for speed
    sample = random.sample(train_data, min(2000, len(train_data)))

    for i, ex in enumerate(sample):
        prompt = ex["prompt"]
        gt     = ex["answer"]
        task   = ex.get("task", "math")

        # Generate K=4 candidates at temperature 0.8 for diversity
        candidates = []
        for _ in range(RAFT_K):
            c = generate(model, tokenizer, prompt,
                          max_tokens=400, temp=0.8, verbose=False)
            candidates.append(c)

        # Score each candidate: ORM + format
        scores = [orm_score(c, gt, task) + format_reward(c) for c in candidates]
        best_idx   = scores.index(max(scores))
        best_score = scores[best_idx]

        # Only keep examples where at least one candidate was correct
        if best_score >= 1.0:  # ORM reward = 1.0 means correct answer
            raft_winners.append({
                "prompt":     prompt,
                "completion": candidates[best_idx],
                "answer":     gt,
                "task":       task,
                "source":     f"raft_iter{iteration}",
                "reward":     best_score,
            })
            stats["correct_any"] += 1
            if orm_score(candidates[0], gt, task) == 1.0:
                stats["correct_k1"] += 1

        stats["total"] += 1

        if (i+1) % 100 == 0:
            pass_at_1   = stats["correct_k1"]  / stats["total"] * 100
            pass_at_k   = stats["correct_any"] / stats["total"] * 100
            print(f"  [{i+1}/{len(sample)}] pass@1={pass_at_1:.1f}% pass@K={pass_at_k:.1f}%"
                  f" winners={len(raft_winners)}")

    # Save winners as the fine-tuning dataset for this iteration
    winners_path = f"data/processed/raft_winners_iter{iteration}.jsonl"
    with open(winners_path, "w") as f:
        for w in raft_winners:
            f.write(json.dumps(w) + "\n")

    print(f"\n  Winners saved: {len(raft_winners)} / {len(sample)}")
    print(f"  Pass@1 baseline: {stats['correct_k1']/stats['total']*100:.1f}%")
    print(f"  Pass@K (K={RAFT_K}): {stats['correct_any']/stats['total']*100:.1f}%")

    # Fine-tune on winners using mlx-lm LoRA
    prev_ckpt = f"checkpoints/sft" if iteration == 1 else f"checkpoints/raft/iter{iteration-1}"
    curr_ckpt = f"checkpoints/raft/iter{iteration}"
    Path(curr_ckpt).mkdir(parents=True, exist_ok=True)

    # Create symlink for val data
    import os
    os.makedirs(f"data/raft_iter{iteration}", exist_ok=True)
    import shutil
    shutil.copy(winners_path, f"data/raft_iter{iteration}/train.jsonl")
    shutil.copy("data/splits/valid.jsonl", f"data/raft_iter{iteration}/valid.jsonl")

    print(f"\n  Fine-tuning on {len(raft_winners)} winners...")
    cmd = [
        "python", "-m", "mlx_lm.lora",
        "--model", BASE_MODEL,
        "--resume-adapter-file", prev_ckpt + "/adapters.safetensors",
        "--train",
        "--data", f"data/raft_iter{iteration}",
        "--batch-size", "1",
        "--iters", "1000",
        "--learning-rate", "5e-6",
        "--lora-rank", "16",
        "--num-layers", "16",
        "--adapter-path", curr_ckpt,
        "--max-seq-length", "400",
    ]
    subprocess.run(cmd, check=True)
    print(f"\n✅ RAFT iteration {iteration} complete → {curr_ckpt}")
    return curr_ckpt

# ── Main loop ─────────────────────────────────────────────────
if __name__ == "__main__":
    ap = argparse.ArgumentParser()
    ap.add_argument("--start-iter", type=int, default=1)
    args = ap.parse_args()

    # Load from last checkpoint before starting
    start_adapter = "checkpoints/sft" if args.start_iter == 1 \
                   else f"checkpoints/raft/iter{args.start_iter-1}"

    print(f"Loading model with adapter: {start_adapter}")
    model, tok = load(BASE_MODEL, adapter_path=start_adapter)

    with open("data/splits/train.jsonl") as f:
        train_data = [json.loads(l) for l in f]
    # Use only math examples for RAFT (rule-based ORM is most accurate here)
    train_math = [ex for ex in train_data if ex.get("task") == "math"]
    print(f"Math training examples for RAFT: {len(train_math)}")

    for it in range(args.start_iter, RAFT_ITERS + 1):
        new_ckpt = run_raft_iteration(it, model, tok, train_math)
        # Reload model with new adapter for next iteration
        del model, tok
        model, tok = load(BASE_MODEL, adapter_path=new_ckpt)
        # Quick eval after each iteration
        print(f"\nRunning quick eval after iteration {it}...")
        subprocess.run(["python", "scripts/05_eval.py",
                         "--adapter", new_ckpt, "--n", "50"])

    print("\n🎉 All RAFT iterations complete!")
    print(f"Final model: checkpoints/raft/iter{RAFT_ITERS}")
    print("Run full benchmark: python scripts/07_final_benchmark.py")
